YellingMan — 设计规范

技术栈（匹配课程实验指南）

  • Android Java（不使用游戏引擎）
  • 使用 SurfaceView + Canvas 进行渲染
  • 游戏循环使用 MainThread extends Thread
  • 游戏对象采用 CharacterSprite 模式
  • 使用 AudioRecord 处理麦克风输入
  • 无后端，本地使用 SQLite 存储分数

核心机制：语音音量 → 跳跃力度

提取自原始 JumpLevel.java：

SAMPLE_RATE  = 8000 赫兹
ENCODING     = PCM 16位
BUFFER_SIZE  = AudioRecord.getMinBufferSize(8000, CHANNEL_IN_DEFAULT, PCM_16BIT)

循环（约每50毫秒）：
  1. 从麦克风读取 BUFFER_SIZE 个短整型数据到 buffer[]
  2. v = 所有采样值的 buffer[i]^2 之和
  3. 平均值 = v / 读取的采样数
  4. 若 平均值 >= 1:  音量(分贝) = 10 * log10(平均值)
     否则:           音量 = 0
  5. 将分贝值映射到跳跃等级（示例，实际会根据用户个性定制化）：
       音量 < 80    → 0级（不跳跃）
       80 ≤ 音量 < 82 → 1级（小跳）
       82 ≤ 音量 < 85 → 2级（中跳）
       音量 ≥ 85      → 3级（大跳）

游戏规则（GameScreen.java）

  • 角色以恒定速度自动向右奔跑
  • 玩家通过语音音量控制跳跃高度
  • 仅当角色站在平台上时允许跳跃（地面检测）
  • 收集物：星星/爱心用于计分
  • 死亡条件：掉落至地图底部 → 游戏结束；触碰障碍物（火焰）→ 游戏结束；速度接近0（被卡住）→ 游戏结束
  • 胜利条件：到达地图右边缘 → 关卡完成

while (running):
    canvas = surfaceHolder.lockCanvas()
    synchronized (surfaceHolder):
        gameView.update()    // 物理运算、碰撞检测、状态转换
        gameView.draw(canvas) // 渲染当前状态
    surfaceHolder.unlockCanvasAndPost(canvas)

关卡布局格式

每个关卡定义如下：

class LevelData {
    int mapWidth;         // 瓦片数（原始：1000，我们使用更小值如200）
    int tileSize;         // 每个瓦片的像素数（原始：30x36）
    int[][] floorTiles;   // 地面层的行优先瓦片网格
    int[] starPositions;  // 可收集星星的(x, y)坐标对
    int[] obstaclePositions; // 障碍物的(x, y)坐标对
    float gravity;        // 世界重力
    float playerSpeed;    // 自动奔跑速度
}

关卡设计结构：开场平坦跑道 → 第一个障碍物 → 跳跃间隙 → 星星群 → 更难的障碍物 → 终点线。

碰撞检测（不使用Box2D — 简单矩形运算）

// 玩家与星星碰撞
if (rectOverlap(player.x, player.y, player.w, player.h, star.x, star.y, star.w, star.h))
    collectStar()

// 玩家与障碍物碰撞
if (rectOverlap(player, obstacle))
    triggerDeath()

// 玩家与地面碰撞
for each floorTile:
    if (player standing on top of tile)
        player.isGrounded = true

// 玩家掉落
if (player.y > mapHeight) triggerDeath()

// 玩家到达终点
if (player.x > mapWidth * tileSize) triggerVictory()

语音校准（优化特性1）

问题：固定的分贝阈值（80、82、85）并非适用于所有人。轻声的人可能永远达不到1级；大声的人可能始终触发3级。

解决方案：首次启动时，在主菜单前显示一次性校准界面。

校准流程：
  1. "请正常说话3秒钟..." → 记录最小/最大分贝值
  2. "现在尽可能大声喊！" → 记录峰值分贝值
  3. 存储：quietDb（最小值），loudDb（最大值）
  4. 按比例映射阈值：
       范围 = loudDb - quietDb
       1级阈值 = quietDb + 范围 * 0.3
       2级阈值 = quietDb + 范围 * 0.6
       3级阈值 = quietDb + 范围 * 0.85

实现方式：

  • SoundMeter.java 将 quietDb 和 loudDb 存储在 SharedPreferences 中
  • 若 SharedPreferences 中无校准数据 → 显示 CalibrationScreen 状态
  • 校准完成后，getJumpLevel() 使用按比例映射的阈值而非固定数值
  • 设置中的"重新校准"按钮可重置存储的数值

设计价值：解决了实际的用户体验问题。体现对可访问性和边缘场景的考量，而非仅关注理想情况。

连击计分（优化特性2）

问题：收集星星仅给予固定分数。缺乏持续良好表现的激励机制。

解决方案：连续收集星星且不遗漏会累积连击乘数。遗漏一颗星星则重置乘数。

combo = 0
comboTimer = 2.0 (秒)

每帧更新：comboTimer -= dt
收集到星星时：
    combo++
    comboTimer = 2.0
    score += starBaseValue * combo
    显示"x{combo}"浮动文本

若 comboTimer 归0：
    combo = 0
    comboTimer = 2.0

实现方式：

  • GameView 新增两个字段：int comboCount、float comboTimer
  • update() 递减 comboTimer；若 ≤0，重置 comboCount
  • 收集星星时：comboCount++、comboTimer = 2.0f、score += 100 * comboCount
  • HUD.draw() 在连击数>1时，在分数旁显示"x2"/"x3"
  • 浮动连击文本：小型 ComboText 精灵，0.5秒内向上淡出

设计价值：将星星收集从"清单式任务"转变为"技能型系统"。连击+语音控制要求玩家保持稳定的发声强度以最大化分数。

屏幕震动（优化特性3）

问题：游戏缺乏直观的反馈感。满级音量的大喊应带来强烈的体感。

解决方案：当玩家以3级音量大喊时，整个游戏画布在数帧内随机小幅偏移，然后恢复居中。

shakeDuration = 0   (剩余帧数，0=无震动)
shakeIntensity = 5  (像素，最大偏移量)

当起跳时 jumpLevel == 3：
    shakeDuration = 8   (8帧 ≈ 60fps下的130毫秒)

每帧绘制时：
    if shakeDuration > 0:
        offsetX = random(-shakeIntensity, +shakeIntensity)
        offsetY = random(-shakeIntensity, +shakeIntensity)
        canvas.translate(offsetX, offsetY)
        shakeDuration--

实现方式：

  • GameView 新增两个字段：int shakeFrames、float shakePower
  • 触发3级跳跃时设置 shakeFrames = 8
  • 在 draw() 中，渲染游戏世界前，若 shakeFrames > 0 则应用 canvas.translate() 随机偏移
  • 强度渐变：currentPower = shakePower * (shakeFrames / 8.0f) 实现平滑衰减

设计价值：对游戏手感/体验细节的理解。感知效果提升显著。注重"打磨度"和"细节关注"。

精灵动画

遵循实验指南的 CharacterSprite 模式：

  • 使用 Bitmap.createBitmap() 将精灵图分割为动画帧
  • 动画以固定速率循环播放帧（如每帧1/12秒）
  • CharacterSprite.update() 处理位置 + 速度
  • CharacterSprite.draw(Canvas) 绘制当前帧

数据持久化

  • 使用SQLite数据库存储高分
  • 数据表：scores(_id, level, score, date)
  • 通过 ScoreDBHelper extends SQLiteOpenHelper 实现简单的CRUD操作

关卡数据格式定义

class LevelData {
    int mapWidthTiles;     // 总宽度（瓦片单位）
    int tileSize;          // 每块瓦片像素（64px）
    int floorY;            // 地面像素Y坐标

}

关卡设计规律

所有关卡遵循统一节奏：平地开场 → 首个障碍 → 跳跃坑 → 星星集群 → 高难度障碍 → 终点冲刺
