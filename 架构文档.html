游戏循环（MainThread）

while (running):
    canvas = surfaceHolder.lockCanvas()
    synchronized (surfaceHolder):
        gameView.update()      // 帧率 60 帧/秒
        gameView.draw(canvas)
    surfaceHolder.unlockCanvasAndPost(canvas)

update() 分发到当前状态处理逻辑。只有 PLAYING（游戏中） 状态每帧执行物理、碰撞、计分逻辑；其他状态为空闲或定时器驱动。

游戏中状态 — 每帧更新流程

  1. 从 SoundMeter 获取跳跃等级
  2. 角色在地面 且 跳跃等级>0 → 执行对应等级跳跃
  3. 跳跃等级为3 → 开启10帧屏幕震动
  4. 更新角色物理（重力、速度、动画）
  5. 检测地面碰撞（射线检测角色下方瓦片）
  6. 检测星星碰撞（矩形相交 → 收集、连击+1）
  7. 检测障碍碰撞（矩形相交 → 角色死亡、切换状态）
  8. 检测：角色Y坐标超过地图高度 → 死亡
  9. 检测：角色X坐标超过终点瓦片 → 胜利（保存分数、更新进度）
  10. 更新相机偏移（地图滚动）
  11. 更新连击计时器 / 提示计时器 / 震动效果

语音处理流程

麦克风（8000Hz，16位PCM）
    │
    ▼
AudioRecord.read() ──► short[] 音频缓冲区
    │
    ▼
计算RMS → 分贝 = 10 * log10(平均值) （SoundMeter线程，每50ms执行）
    │
    ▼
读取校准阈值（来自SharedPreferences）
    │  阈值1 = 安静分贝 + 范围 * 0.30
    │  阈值2 = 安静分贝 + 范围 * 0.60
    │  阈值3 = 安静分贝 + 范围 * 0.85
    ▼
跳跃等级：0=静音  1=小声  2=中声  3=大声
    │
    ▼
Player.jump(等级):
    垂直速度 = 跳跃速度数组[等级]     {0, -420, -620, -820}
    标记为非地面状态

校准流程

  1. 阶段0："请用正常音量说话"（3秒）→ 记录正常说话最低分贝
  2. 阶段1："请大声喊叫！"（3秒）→ 记录大喊最高分贝
  3. 阶段2："校准完成！"（2秒）→ 计算阈值、保存到本地 → 自动进入主菜单

每次启动都会执行校准，适配环境噪音变化。

分数与连击系统

  • 单颗星星基础分：100 分
  • 连击规则：2秒内连续收集星星触发倍数加成
  • 分数计算：100 × 连击数
  • 连击重置：超过2秒未收集星星，连击清零
  • 特效：收集星星位置显示浮动"x3!"文字（0.6秒渐隐）
  • 分数保存：仅通关胜利时保存分数，中途死亡不保存

数据持久化

存储内容 存储方式 用途
安静分贝、大喊分贝、校准状态 SharedPreferences（yellingman_cal） 语音校准阈值
最高通关关卡 SharedPreferences（yellingman_progress） 开始按钮默认目标关卡
分数表 SQLite（yellingman_scores.db） 各关卡最高分记录

屏幕震动效果

  • 触发条件：跳跃等级为3（大喊）起跳时
  • 持续时间：10帧（约167毫秒）
  • 震动强度：最大6像素，线性衰减
  • 实现方式：绘制时通过 canvas.translate 随机偏移画布
